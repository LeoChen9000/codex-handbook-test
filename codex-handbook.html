第一部分：认识 Codex 入门

1.1 什么是 Codex？

Codex 是 OpenAI 推出的编程智能体（Coding Agent），专为软件开发而设计。它能够：

  • 编写代码 — 根据自然语言描述生成完整功能
  • 理解代码 — 阅读和分析整个代码库
  • 调试修复 — 定位并修复 bug
  • 重构优化 — 改善代码质量和架构
  • 运行测试 — 执行测试并分析结果
  • 多代理协作 — 并行派遣多个子代理处理不同任务
💡 核心概念：Codex 不是简单的代码补全工具。它是一个自主代理，可以在你的工作区中执行命令、读写文件、搜索代码、运行测试，像一个真正的结对程序员一样与你协作。

1.2 Codex 的使用场景

场景说明
新项目脚手架输入需求，自动生成项目结构和样板代码
Bug 修复描述错误现象，Codex 定位问题并修复
功能开发从需求到实现，一步到位
代码审查让 Codex 审查代码质量、安全性、性能
文档生成自动为代码库生成文档
学习探索快速理解陌生代码库的结构和逻辑

1.3 Codex 的三个使用面

  • Codex CLI — 终端命令行界面，适合服务器环境和自动化流程
  • Codex Desktop App — 桌面应用程序，带图形界面，适合日常开发
  • Codex API — 编程接口，可嵌入你自己的应用中

第三部分：基础使用入门 入门

3.1 第一次对话

打开 Codex 后，直接在输入框或终端中输入你的需求即可。例如：

# 创建一个 Express.js 项目
帮我创建一个 Express.js 项目，包含用户注册、登录、JWT 认证中间件

# 修复一个 bug
这个函数在处理空数组时会崩溃，帮我修复：
function sum(arr) {
  return arr.reduce((a, b) => a + b);
}

3.2 基本交互流程

1
描述问题 — 用自然语言清楚描述你想做什么
2
Codex 分析 — Codex 会先浏览相关代码文件，理解上下文
3
制定计划 — Codex 会生成一个执行计划，显示它将分几步完成
4
执行操作 — Codex 开始读写文件、运行命令、执行测试
5
结果反馈 — Codex 完成后汇报做了什么，你审阅并给出反馈

3.3 高效提问的技巧

❌ 低效提问✅ 高效提问
"帮我写个程序""帮我写一个 Python 脚本，遍历当前目录统计所有 .py 文件的行数"
"有 bug""运行 npm test 时第三个测试失败，错误信息是 TypeError: undefined is not a function"
"优化代码""把 src/auth.js 中的同步文件读取改为异步，提升性能"
✅ 黄金法则：描述越具体，Codex 的输出越精准。尽量提供：文件路径、错误信息、期望行为、技术栈偏好。

第五部分：桌面应用使用指南 入门

5.1 界面概览

Codex 桌面应用包含以下主要区域：

  • 左侧边栏 — 会话列表、线程管理、历史记录
  • 中央面板 — 聊天交互区，输入需求、查看 Codex 回复
  • 右侧面板 — 文件浏览器、代码预览、终端输出
  • 底部状态栏 — Token 用量、模型信息、连接状态

5.2 创建新线程

每个线程代表一个独立的任务上下文。建议：

  • 每个功能开发创建一个新线程
  • Bug 修复和特性开发分开线程
  • 复杂任务可以分叉线程进行并行探索

5.3 线程操作

  • 重命名 — 右键点击线程，选择"重命名"，便于后续查找
  • 固定 — 将重要线程置顶，方便快速访问
  • 分叉 — 基于现有线程创建分支，保留历史上下文
  • 归档 — 完成的任务归档到历史，保持侧边栏整洁

5.4 文件操作

Codex 桌面应用可以直接：

  • 在工作区中创建、编辑、删除文件
  • 通过 apply_patch 精确修改代码片段
  • 在右侧面板实时预览文件内容
  • 查看 Git 差异对比
📌 提示：Codex 使用 apply_patch 而非全文替换的方式修改文件，这种方式更安全、更精确，且能保留 Git 的变更追踪。

第六部分：Skills 技能系统 进阶

6.1 什么是 Skills？

Skill（技能）是 Codex 的可扩展能力模块。每个 Skill 由一个 SKILL.md 文件定义，告诉 Codex 在特定场景下应该如何行动。就像给 Codex 安装"专业证书"。

6.2 内置 Skills

Skill用途
imagegen图像生成与编辑，支持 AI 创建位图视觉素材
openai-docs查询 OpenAI 官方文档，获取最新 API 和模型信息
plugin-creator创建和编排 Codex 插件目录结构
skill-creator指导如何创建新的 Skill
skill-installer从社区仓库安装第三方 Skill
find-skills发现和推荐合适的 Skill 供安装

6.3 安装 Skill

# 安装社区 Skill
npx skills add vercel-labs/skills@find-skills -g -y

# 从 GitHub 仓库安装
npx skills add github-user/repo@skill-name -g -y

# 安装后重启 Codex 生效

6.4 自定义 Skill

你可以为自己的项目创建专属 Skill。在 .codex/skills/ 目录下创建：

.codex/skills/
└── my-project/
    ├── SKILL.md          # 技能定义文件
    └── references/       # 参考资料目录
        ├── api-guide.md
        └── conventions.md

SKILL.md 的基本格式：

---
name: "my-skill"
description: "当用户需要...时使用此技能"
---

# 我的技能名称

## 触发条件
当用户提到 XXX 时，使用此技能。

## 执行步骤
1. 第一步做什么
2. 第二步做什么
3. ...

## 注意事项
- 注意事项 1
- 注意事项 2
💡 最佳实践：Skill 的 description 字段决定了 Codex 何时自动调用它。写得越准确，Codex 识别得越及时。

第八部分：MCP 集成 高阶

8.1 什么是 MCP？

MCP（Model Context Protocol）是 OpenAI 定义的标准化协议，用于让 Codex 与外部工具和服务通信。通过 MCP，Codex 可以：

  • 访问远程 API 和数据源
  • 调用外部服务（如数据库、搜索引擎）
  • 集成第三方工具链

8.2 配置 MCP 服务器

# 添加 OpenAI 开发者文档 MCP 服务器
codex mcp add openaiDeveloperDocs --url https://developers.openai.com/mcp

# 添加自定义 MCP 服务器
codex mcp add myServer --url http://localhost:3000/mcp

8.3 查看可用 MCP 资源

# 列出所有 MCP 资源
codex mcp list-resources

# 列出资源模板
codex mcp list-templates

第九部分：Prompt 编写技巧 进阶

9.1 Prompt 分层结构

高效的 Codex 使用涉及三个层次的指令：

层级文件作用域优先级
最高层用户直接输入当前对话最高
中间层AGENTS.md所在目录及子目录中高
底层SKILL.md全局中

9.2 AGENTS.md 使用指南

在项目根目录创建 AGENTS.md 文件，为 Codex 提供项目级别的定制指令：

# AGENTS.md

## 编码规范
- 使用 TypeScript 严格模式
- 文件名使用 camelCase
- 组件使用函数式写法

## 技术栈偏好
- 前端：React 18 + TypeScript
- 后端：Node.js + Express
- 数据库：PostgreSQL

## 特殊说明
- 不要修改 src/config/ 下的文件
- 测试文件放在 tests/ 目录下
⚠️ 注意：AGENTS.md 的作用域是其所在目录及其所有子目录。更深层级的 AGENTS.md 优先于外层文件。

9.3 高效 Prompt 模板

模板一：新功能开发

请在 src/ 目录下实现 [功能名称]。

要求：
1. 使用 [技术栈]
2. 遵循项目的 [编码规范]
3. 编写对应的单元测试
4. 更新 README 中的相关说明

参考文件：[列出已有类似功能的文件路径]

模板二：Bug 修复

发现一个 Bug：
- 文件：src/auth/login.ts
- 第 [XX] 行
- 错误信息：[粘贴错误]
- 复现步骤：
  1. [步骤一]
  2. [步骤二]

预期行为：[描述应该发生什么]
实际行为：[描述实际发生了什么]

模板三：代码审查

请审查 src/payment/ 目录下的代码，重点关注：
1. 安全性问题（SQL 注入、XSS 等）
2. 性能瓶颈
3. 代码重复
4. 缺少错误处理的地方

输出格式：按文件列出发现的问题和建议

模板四：重构优化

请将 src/utils/helpers.js 重构为 TypeScript。

要求：
- 添加类型定义
- 提取公共逻辑到独立模块
- 保持现有 API 不变
- 更新所有引用该文件的导入路径

第十一部分：多代理协作 高阶

11.1 什么是多代理？

Codex 可以派遣子代理（Sub-Agents）并行处理多个子任务。这就像组建一个虚拟开发团队，每个成员负责不同的模块。

11.2 适用场景

  • 大型重构 — 不同模块由不同代理并行处理
  • 多文件修改 — 各代理负责不同文件的更新
  • 并行探索 — 同时研究多种技术方案

11.3 使用方式

# 在对话中描述需要并行处理的任务
请同时完成以下工作：
1. 为用户模块编写单元测试
2. 为订单模块编写集成测试
3. 为支付模块编写 E2E 测试

这三个任务互不影响，可以并行执行。

11.4 代理类型

类型用途特点
默认代理通用任务继承父代理模型和能力
Explorer 代理代码库探索快速、权威，擅长回答特定问题
Worker 代理代码修改执行负责具体的编码和修改工作
⚠️ 注意事项：并行代理会增加 token 消耗。对于简单任务，直接让主代理处理更高效。多代理适合大规模、独立性强的子任务。

第十四部分：安全与最佳实践 进阶

14.1 密钥安全管理

🔒 安全警告：永远不要将 API 密钥提交到版本控制系统！ 
# 确保密钥不在 Git 历史中
# 添加到 .gitignore
echo ".codex/config.json" >> .gitignore

# 使用环境变量存储密钥（推荐）
# Windows PowerShell
[System.Environment]::SetEnvironmentVariable(
  "OPENAI_API_KEY", "[你的密钥]", "User"
)

# Linux / macOS
echo 'export OPENAI_API_KEY="[你的密钥]"' >> ~/.bashrc

14.2 文件操作安全

第十五部分：常见问题 FAQ 入门

Q1：Codex 和 GitHub Copilot 有什么区别？

Copilot 是代码补全工具，主要在编辑器内提供单行/单函数的代码建议。Codex 是完整的编程智能体，可以在整个项目层面工作：读文件、运行命令、执行测试、管理 Git。Copilot 像是"键盘助手"，Codex 更像是"结对程序员"。

Q2：Codex 会泄露我的代码吗？

OpenAI 承诺不会使用 Codex 的交互数据来训练模型。但出于安全考虑，建议不要在 prompt 中发送敏感信息（密码、密钥、个人数据）。

Q3：Codex 支持哪些编程语言？

Codex 支持几乎所有主流编程语言，包括但不限于：JavaScript/TypeScript、Python、Go、Rust、Java、C/C++、Ruby、PHP、Swift、Kotlin 等。它的代码质量取决于你提供的上下文清晰度。

Q4：如何处理 Codex 生成的错误代码？

# 1. 将错误信息直接发给 Codex
这个代码运行时报错：[粘贴错误]
请修复。

# 2. 让它运行测试看结果
请运行 npm test，分析失败的测试并修复。

# 3. 提供更多信息
在 src/auth.js:42 行附近有问题，
错误是 TypeError: xxx is not a function。

Q5：Codex 的 token 消耗怎么计算？

Token 消耗取决于：

  • 输入上下文的文件大小和数量
  • Codex 执行的命令数量和复杂度
  • 代码生成的长度
  • 多代理并行时的额外开销

建议在 config.json 中设置 token 预算以避免意外高额费用。

Q6：Codex 能帮我做代码审查吗？

完全可以！使用方法：

请审查以下变更：
1. 安全性：是否有 SQL 注入、XSS 等漏洞？

4. 测试：是否有对应的单元测试？

重点关注 src/payment/ 目录的改动。

Q7：Codex 可以集成到我的 CI/CD 流程吗？

可以。Codex CLI 支持非交互模式，可以在 CI 管道中使用：

- name: Auto test generation
  run: |

         --output tests/

Q8：如何提升 Codex 的回答质量？

  1. 提供上下文 — 发送相关代码片段或文件路径
  2. 明确期望 — 说清楚你想要什么，而不是"随便写写"
  3. 迭代 refinement — 不满意就继续描述，Codex 会逐步改进
  4. 使用 AGENTS.md — 固化项目规范和偏好，让 Codex 每次都记住
  5. 指定模型 — 复杂任务用 gpt-5.5，简单任务用 gpt-5.4-mini

案例十：文件整理 — 智能分类与归档

3. 按文件大小分布（小于1MB / 1-10MB / 10-100MB / 大于100MB）

5. 找出重复文件（按文件名或MD5哈希）

   - 图片：.jpg .png .gif .svg .webp → images/
   - 文档：.pdf .doc .docx .txt → documents/
   - 表格：.xls .xlsx .csv → spreadsheets/
